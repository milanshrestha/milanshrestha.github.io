## 3. **Architecture Overview**

### **3.1. Content Architecture**

* Markdown files (`.md`) represent content.
* Each file uses front matter (`YAML` block) to define title, layout, and metadata.

Example â€” `projects/wifi-automation.md`:

```markdown
---
title: "Guest WiFi Automation with FreeRADIUS & daloRADIUS"
layout: "project.njk"
date: 2025-03-15
tags: ["project", "network", "automation"]
summary: "Automated guest WiFi access system integrated with PMS."
---

A fully automated guest WiFi authentication solution using FreeRADIUS, integrated with hotel PMS.
```

### **3.2. Layout Architecture**

* `base.njk` â€” wraps all pages (includes header, footer, SEO tags).
* `home.njk` â€” custom layout with hero intro, project grid.
* `page.njk` â€” generic layout for About, Resume, Contact.

### **3.4. Styling System**

* **CSS Architecture:**

  * BEM-style naming
  * Component-based partials
  * Variables for colors, fonts, spacing
  * Compiled using Sass

* **Theme Guidelines:**

  * Typography: `Google Sans` (headings), `EB Garamond` (body)
  * Primary Color: `#1A73E8`
  * Accent: `#202124`
  * Neutral: `#F8F9FA`
  * Card Shadows, rounded corners, fluid spacing

### **3.6. Collections Setup (Inside `.eleventy.js`)**

Example:

```js
module.exports = function(eleventyConfig) {
  eleventyConfig.addPassthroughCopy("src/assets");
  eleventyConfig.addCollection("projects", (collection) => {
    return collection.getFilteredByGlob("src/projects/*.md");
  });
  eleventyConfig.addCollection("posts", (collection) => {
    return collection.getFilteredByGlob("src/posts/*.md");
  });
  return {
    dir: {
      input: "src",
      output: "dist",
      includes: "_includes",
      data: "_data"
    }
  };
};
```
